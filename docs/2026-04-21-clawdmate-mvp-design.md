> 创建日期：2026-04-21

---

## 一、产品定位

ClawdMate 是一个通过「灵动岛时间进度 + 小螃蟹陪伴 + 轻提醒」，在不打扰心流的情况下，帮助用户控制学习节奏的陪伴工具。

**核心价值**：陪伴 + 时间感知，而非任务管理。

**核心关键词**：学习陪伴 | 时间控制 | 无打扰提醒 | 过程记录

---

## 二、技术架构

| 层级 | 技术选型 |
|------|----------|
| 桌面端 | Tauri（Rust 核心） |
| 前端 | React + TypeScript |
| UI | Tailwind CSS + Framer Motion |
| 后端 | Supabase（认证 + 数据库 + 实时同步） |
| 音效 | AI 生成，俏皮可爱风（已确定，迭代实现） |
| 宠物动画 | CSS/SVG 帧动画（已确定，迭代实现） |

---

## 三、系统组成

**两个端：**

1. **Windows 桌面端**（Tauri）—— 灵动岛 + 小螃蟹
2. **Web 端**（React SPA）—— Landing Page + 功能面板

---

## 四、MVP 功能范围

**目标**：完成最核心的「输入任务 → 灵动岛计时 → 完成」流程

| 功能 | 描述 |
|------|------|
| 任务输入 | 极简模式（任务+时间）或结构模式（子任务+顺序） |
| 灵动岛 | 桌面端悬浮进度条，收起/展开两种状态 |
| 多任务并行 | 支持多个任务同时计时，灵动岛可切换显示 |
| 时间控制 | 暂停、结束、延长时间（+5/+10/+15/自定义） |
| 轻提醒 | 文字提示 + 关键节点通知（时间过半、剩5分钟、时间到） |

## 六、桌面端设计 — 灵动岛

### 6.1 位置与形态

- **位置**：屏幕顶部居中
- **形态**：紧凑悬浮条，始终置顶
- **交互**：点击展开查看详情

### 6.2 收起状态

```
┌─────────────────────────────────────────┐
│  🦀  [██████████░░░░░░]  学习 React    │
└─────────────────────────────────────────┘
```

- 时间进度条
- 当前任务名称
- 小螃蟹（始终可见，迭代二实现）

### 6.3 展开状态

```
┌─────────────────────────────────────────────────┐
│                                         🦀      │
│   当前任务：学习 React                           │
│   进度：45%  剩余：25/60 分钟                    │
│                                                 │
│   [暂停]  [结束]                                │
│                                                 │
│   ─── 并行任务 ───                              │
│   ○ 写作业      [███░░░]  暂停中  15/30分钟     │
│   ○ 复习英语    [██████]  已完成  20分钟        │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 当前任务详情（名称、进度、剩余时间）
- 并行任务列表（显示进度条 + 状态 + 时间，可切换活跃任务）
- 操作按钮
- 小螃蟹动画（迭代二实现）

### 6.4 小螃蟹 7 状态动画（迭代二实现）

| 状态 | 触发时机 | 动画表现 |
|------|----------|----------|
| **Idle** | 无任务时 | 轻微呼吸、眨眼、左右看看 |
| **Starting** | 点击开始 | 跳一下、举手、「加油」小气泡 |
| **Focusing** | 任务进行中 | 坐着看书/敲键盘、偶尔写字、偶尔点头 |
| **Reminder** | 时间节点 | 抬头、挥手、小铃铛/感叹号闪现 |
| **Overtime** | 超过预设时间 | 有点着急、举牌子、小表情变成「诶？」 |
| **Done** | 任务完成 | 跳跃、转圈、撒星星 |
| **Resting** | 选择休息 | 躺下、打哈欠、睡觉冒泡泡 |

### 6.5 提醒机制

**MVP 实现**：

1. **进度条内文字提示**：灵动岛内文字变化，不额外弹窗
2. **关键节点通知**：时间过半、剩 5 分钟、时间到等

**迭代二实现**：

3. **俏皮音效**：AI 生成的可爱提示音
4. **小螃蟹微动画**：状态变化时触发

### 7.2 游客模式（迭代一）

- 无需登录即可使用所有核心功能
- 数据仅存储在浏览器本地（localStorage）
- 登录后可同步数据到云端

### 7.3 任务填写模块

**两种模式**：

1. **极简模式**：输入任务 + 预设时间 → 直接开始
2. **结构模式**：输入任务 + 子任务列表 → 按顺序进行

```
┌─────────────────────────────────────┐
│  新建任务                           │
│                                     │
│  任务名称：[________________]       │
│                                     │
│  预设时间：[25] 分钟                │

│  □ 添加子任务                       │
│                                     │
│      [开始学习]  🦀                 │
└─────────────────────────────────────┘
```

## 八、核心交互流程

```
用户打开 Web/桌面端
    │
    ▼
Landing Page（迭代四）
    │
    ├── 登录 → 进入功能面板（数据云同步）
    │
    └── 直接进入 → 功能面板（游客模式，数据仅本地）
    │
    ▼
输入任务 + 设定时间
    │
    ▼
点击开始 ──────► 小螃蟹 Starting 状态（迭代二）
    │
    ▼
灵动岛出现（桌面端，MVP）
    │
    ▼
小螃蟹进入 Focusing 状态（迭代二）
    │
    ├── 时间节点 ──► 文字提示 + Reminder 动画（迭代二）+ 音效（迭代二）
    │
    ▼
时间到达
    │
    ├── 选择延长 ──► 继续计时
    │                │
    │                ▼
    │             超时 ──► Overtime 状态（迭代二）
    │
    └── 选择结束 ──► Done 状态（迭代二）+ 庆祝音效（迭代二）
                     │
                     ▼
                  选择下一步
                     │
                     ├── 开始下一个任务
                     │
                     └── 休息 ──► Resting 状态（迭代二）
```

## 九、数据模型

### 9.1 用户表 (users)

```sql
users
  - id: uuid (primary key)
  - email: string
  - password_hash: string
  - created_at: timestamp
  - updated_at: timestamp
```

### 9.2 任务表 (tasks)

```sql
tasks
  - id: uuid (primary key)
  - user_id: uuid (foreign key, nullable)  -- 游客为 null
  - title: string
  - mode: enum ('simple', 'structured')
  - status: enum ('pending', 'active', 'paused', 'completed', 'cancelled')
  - planned_duration: integer (minutes)
  - actual_duration: integer (minutes)
  - started_at: timestamp
  - completed_at: timestamp
  - created_at: timestamp
```

### 9.3 子任务表 (subtasks)

```sql
subtasks
  - id: uuid (primary key)
  - task_id: uuid (foreign key)
  - title: string
  - order: integer
  - status: enum ('pending', 'active', 'completed')
  - started_at: timestamp
  - completed_at: timestamp
  - duration: integer (seconds)
```

### 9.4 每日记录表 (daily_records)

```sql
daily_records
  - id: uuid (primary key)
  - user_id: uuid (foreign key, nullable)  -- 游客为 null
  - date: date
  - total_duration: integer (seconds)
  - task_count: integer
  - created_at: timestamp
```
