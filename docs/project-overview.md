# ClawdMate 项目概述

> 适用于 AI 产品经理简历

---

## 一、项目简介

**ClawdMate** 是一款学习陪伴工具，通过「灵动岛时间进度 + 小螃蟹陪伴 + 轻提醒」的方式，在不打扰用户心流的情况下，帮助用户控制学习节奏、提升专注力。

**核心价值主张**：陪伴 + 时间感知，而非传统任务管理工具。

---

## 二、产品定位

| 维度 | 描述 |
|------|------|
| **目标用户** | 学生、自学者、需要专注工作的知识工作者 |
| **核心场景** | 学习/工作时的专注陪伴、时间控制、过程记录 |
| **差异化优势** | 无打扰提醒 + 情感化陪伴 + 轻量化设计 |
| **产品形态** | 桌面端灵动岛 + Web 端管理面板 |

## 三、技术架构

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      用户界面层                          │
├─────────────────────┬───────────────────────────────────┤
│   Web 端 (React)    │      桌面端 (Tauri + React)       │
│   - 任务管理        │      - 灵动岛悬浮窗               │
│   - 历史记录        │      - 小螃蟹动画                 │
│   - 日报生成        │      - 实时计时                   │
├─────────────────────┴───────────────────────────────────┤
│                    数据同步层                            │
│              (桥接服务 + WebSocket)                      │
├─────────────────────────────────────────────────────────┤
│                    数据存储层                            │
│         (本地存储 / Supabase 云端同步)                   │
└─────────────────────────────────────────────────────────┘
```

### 3.2 技术栈

| 层级 | 技术选型 | 选型理由 |
|------|----------|----------|
| 桌面端框架 | Tauri 2.x | 轻量、高性能、跨平台 |
| 前端框架 | React 18 + TypeScript | 组件化、类型安全 |
| 样式方案 | Tailwind CSS | 快速开发、高度可定制 |
| 动画库 | Framer Motion | 流畅的交互动画 |
| 状态管理 | Zustand | 轻量、简洁、支持持久化 |
| 构建工具 | Vite | 快速热更新、现代化构建 |
| 后端服务 | Supabase | 认证 + 数据库 + 实时同步一体化 |

## 四、核心功能模块

### 4.1 灵动岛（桌面端核心）

| 功能 | 描述 |
|------|------|
| 收起态 | 紧凑悬浮条，显示进度条、剩余时间、任务名称 |
| 展开态 | 完整任务详情、并行任务列表、操作按钮 |
| 多任务并行 | 支持多个任务同时计时，可切换活跃任务 |
| 时间控制 | 暂停、结束、延长时间（+5/+10/+15/自定义） |
| 轻提醒 | 时间过半、剩5分钟、时间到等关键节点通知 |

### 4.2 任务管理（Web 端）

| 模式 | 描述 |
|------|------|
| 极简模式 | 输入任务名称 + 预设时间，快速开始 |
| 结构模式 | 输入任务 + 子任务列表，按顺序逐步完成 |

### 4.3 数据记录

- **今日记录**：当日任务列表、总学习时长、完成情况
- **历史记录**：卡片列表/日历视图切换，查看历史学习记录
- **日报生成**（规划中）：趣味化总结，漫画式日报

### 4.4 双端联动

- Web 端「召唤灵动岛」控制桌面端显示/隐藏
- 任务数据双向同步（Web 创建 ↔ 桌面端同步）
- 状态实时同步（暂停/继续/完成/取消）

## 五、产品亮点

### 5.1 无打扰设计

- 灵动岛始终置顶但不遮挡主工作区
- 提醒通过进度条内文字变化实现，无弹窗打断
- 关键节点通知仅在必要时触发

### 5.2 情感化陪伴

**小螃蟹 7 状态动画**（规划中）：

| 状态 | 触发时机 | 表现 |
|------|----------|------|
| Idle | 无任务时 | 轻微呼吸、眨眼、左右看看 |
| Starting | 点击开始 | 跳跃、举手、「加油」气泡 |
| Focusing | 任务进行中 | 看书/敲键盘、偶尔点头 |
| Reminder | 时间节点 | 挥手、小铃铛闪现 |
| Overtime | 超过预设时间 | 着急、举牌子、「诶？」表情 |
| Done | 任务完成 | 跳跃、转圈、撒星星 |
| Resting | 选择休息 | 躺下、打哈欠、睡觉冒泡泡 |

### 5.3 游客模式

- 无需登录即可使用所有核心功能
- 数据存储在浏览器本地（localStorage）
- 登录后可同步数据到云端

## 八、产品经理视角总结

### 8.1 解决的核心问题

1. **专注被打断**：传统任务工具弹窗提醒打断心流
2. **缺乏陪伴感**：独自学习/工作缺乏情感支持
3. **时间感知弱**：容易忘记时间，导致效率低下

### 8.2 产品创新点

1. **灵动岛形态**：借鉴 iOS 灵动岛设计，信息密度高但不打扰
2. **情感化陪伴**：小螃蟹 IP + 状态动画，增加趣味性和粘性
3. **双端联动**：Web 管理面板 + 桌面灵动岛，覆盖不同使用场景

### 8.3 商业化潜力

- **免费版**：游客模式 + 本地存储
- **付费版**：云端同步 + 多设备 + 高级日报 + 自定义宠物
