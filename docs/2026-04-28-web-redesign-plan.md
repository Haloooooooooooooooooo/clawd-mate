# ClawdMate Web 端复刻执行文档

> 日期：2026-04-28
> 目标：不再做“参考图风格化改编”，而是以 `clawdmate_orange_pixel_v3.html` 为视觉与结构基准，复刻出适配当前桌面屏幕的大屏 web 版本，并把现有任务逻辑映射进去。

## 1. 当前决策

本轮不再做抽象 redesign，也不做自由发挥。

新的执行原则只有一条：

`以参考 HTML 为准，优先复刻布局、颜色、字体、边框、阴影、间距和组件结构，但不照搬它的缩小尺寸，而是扩展成适配当前桌面屏幕的大屏版本。`

这意味着：

- 不再沿用当前 `web` 里混合的像素风 / 新拟物 / 纸张风。
- 不再先做大范围组件设计探索。
- 先把页面外观和结构对齐到参考 HTML，再按大屏工作台方式放大尺寸与留白。
- 参考 HTML 负责定视觉语言，不负责锁死最终尺寸。

## 4. 页面结构基线

参考 HTML 的结构非常清晰，建议直接映射为：

```text
app
├─ sb      左侧栏
├─ main    中间主工作区
└─ rp      右侧信息栏
```

参考 HTML 的原始尺寸是缩小版，因此这里只借用其比例关系，不直接照搬像素宽度。

桌面端布局基线应调整为：

- 左栏：`300px` 到 `340px`
- 右栏：`280px` 到 `320px`
- 中栏：自适应，占据剩余空间
- 中栏内部双列：默认 `minmax(420px, 1fr) minmax(420px, 1fr)`
- 页面主体最大宽度：优先按视口铺开，不再限制成参考稿的小画板
- 页面最小视觉高度：至少铺满当前屏幕高度

当前项目里建议映射为：

- `Sidebar.tsx` 对应 `sb`
- `Dashboard.tsx` 的主体区对应 `main + rp`
- `App.tsx` 继续保留路由壳，但 `/app/dashboard` 进入后应直接呈现三栏复刻布局

## 5. 视觉规范

## 5.1 颜色 token

直接采用参考 HTML 的命名和数值，避免二次猜测：

- `--o: #E07840`
- `--o2: #EE9B6A`
- `--o3: #F5C9A8`
- `--o4: #FBE9DA`
- `--o5: #FEF4EE`
- `--sage: #6B9E7A`
- `--s2: #A8CCAF`
- `--s3: #DFF0E3`
- `--lav: #8A7FBB`
- `--l2: #BDB8D8`
- `--l3: #ECEAF7`
- `--ink: #2C2018`
- `--i2: #6A5240`
- `--i3: #A89080`
- `--i4: #D8C8BC`
- `--bg: #FBF6F0`
- `--bg2: #F5EDE3`
- `--bg3: #EDE0D0`

项目里要做的事：

- 用这些 token 替换当前 `index.css` 里偏泛化的暖色变量
- 保留必要兼容 alias，避免全项目一次性炸开

## 5.2 字体

直接对齐参考 HTML：

- 标题 serif：`Noto Serif SC`
- 正文 sans：`Noto Sans SC`
- 数字 mono：`JetBrains Mono`

应用规则：

- `早上好，Alex`、任务标题用 serif
- 普通文案、导航、按钮用 sans
- 倒计时、统计数字、时长信息用 mono

## 5.3 背景

页面外层背景采用：

- 暖米白底色
- `16px * 16px` 网格
- 网格线颜色使用 `--bg3`

这是整个页面气质的核心，必须优先还原。

## 5.3.1 尺寸放大原则

参考 HTML 是缩小演示稿，正式页面需要整体放大，但保持比例关系。

放大规则：

- 标题字号整体放大约 `1.35x` 到 `1.6x`
- 卡片内边距整体放大约 `1.4x`
- 三栏之间的间距明显增大
- 按钮高度和输入框高度同步增大，避免页面显得“玩具化过小”
- 保留细边框和硬阴影，但不要把边框粗细也按比例无限放大

一句话：

`放大的是版式和可读性，不是把所有像素值机械乘倍数。`

## 5.4 边框与阴影

统一规则：

- 主卡片：`2px solid var(--ink)`
- 次边框：`1px` 或 `1.5px solid var(--o3)`
- 阴影：偏硬朗的错位阴影，不使用模糊大片阴影

主要模式：

- 主卡阴影：`3px 3px 0`
- 按钮阴影：`2px 2px 0` 或 `3px 3px 0`
- 选中 chip 阴影：`1.5px 1.5px 0`

## 5.5 圆角

页面不走大圆角路线。

规则：

- 主体组件统一使用极小圆角或近似直角
- 不再使用当前页面中大量 `16px`、`18px`、`20px` 的柔和圆角

## 7. 代码落地策略

## 7.1 尽量保留逻辑，重写视图和样式

本轮不是重写 store，而是：

- 尽量保留 `useStore`
- 尽量保留任务数据结构
- 尽量保留页面交互逻辑
- 重点重写 JSX 结构和 CSS 类

## 7.2 建议改动文件

本轮核心文件：

- `web/src/index.css`
- `web/src/components/layout/Sidebar.tsx`
- `web/src/pages/Dashboard.tsx`

必要时可新增：

- `web/src/components/dashboard/*`

但如果拆分组件会拖慢复刻速度，可以先在 `Dashboard.tsx` 内完成结构重组，等复刻完成后再考虑拆分。

## 7.3 当前建议优先级

优先把“像”做出来，再做“优雅”。

所以顺序是：

1. 页面骨架像
2. 颜色、字体、边框像
3. 卡片结构像
4. 行为映射正确
5. 最后才是组件拆分和代码美化
